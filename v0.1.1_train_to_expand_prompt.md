# Active Workout Screen — Exercise "Train to Expand" Pattern

## Problem

Currently, when the user selects body parts in the Session Setup screen, **all** exercises for those body parts appear pre-populated with weight/reps/sets from the last session. When "Save Session" is pressed, every exercise with a non-empty weight field gets saved to WEIGHT_TRAINING — even exercises the user didn't actually perform that day.

This means if you select "Quads" and see Squat, Leg Press, Leg Extension, Deadlift Machine — but only actually do Leg Press — all four get recorded. The data becomes inaccurate.

## Solution

Change the exercise display from "all expanded, all saved" to a **"Train to expand"** pattern:

1. Exercises appear as a **collapsed list of names only** with a "Train" button
2. Tapping "Train" **expands** that exercise to show weight/reps/sets fields (pre-filled with last session's numbers for reference)
3. User adjusts numbers → taps **"Complete"** → exercise collapses, marked as "trained" (in-memory only, not yet in DB)
4. Only one exercise expanded at a time — expanding a new one auto-completes the previous
5. "Save Session" writes the SESSIONS row + only the "completed" exercises to WEIGHT_TRAINING
6. Exercises never tapped "Train" on = not saved

## Detailed Behavior

### Exercise List State

Each exercise in the list has three possible visual states:

| State | Appearance | Meaning |
|-------|-----------|---------|
| **Idle** | Exercise name + "Train" button | Not yet interacted with |
| **Expanded** | Exercise name + weight/reps/sets fields + "Complete" button | Currently being edited |
| **Completed** | Exercise name + checkmark + weight/reps/sets summary + "Edit" button | Marked as trained, can re-expand |

### "Train" → Expand

- Tapping "Train" on an idle exercise expands it inline
- Weight/reps/sets fields are pre-filled with the most recent session's values for that exercise (same training style) — this is the existing `getLatestWeightForExercise` lookup
- If no previous data exists, fields are empty
- If another exercise is currently expanded, it auto-completes first (saves its current field values as "completed" state)

### "Complete" → Collapse

- Tapping "Complete" collapses the exercise back to the completed state
- Shows a summary: e.g., "Squat ✓ — 40kg × 12 × 4"
- The exercise is now marked as "trained" for this session
- Data is held **in-memory only** — not written to the database yet

### "Edit" → Re-expand

- Tapping "Edit" on a completed exercise re-expands it with the current values
- User can modify and tap "Complete" again
- This can be done any number of times before "Save Session"

### "Save Session" — What Gets Saved

- The SESSIONS row is created as before (date, workout, body parts, cardio, notes, etc.)
- **Only exercises in the "completed" state** get written to WEIGHT_TRAINING
- Exercises still in "idle" state (never tapped "Train") are **not saved**
- If an exercise is currently expanded when "Save Session" is pressed, auto-complete it first (treat its current field values as completed), then save

### Body Part Selection Changes

- When body parts are changed (chips toggled), the exercise list reloads
- Previously completed exercises for body parts that are still selected should **remain completed** (don't lose their state)
- Exercises for body parts that were deselected are removed from the list
- New exercises added from newly selected body parts start in "idle" state

### Reset Form

- "Reset Form" clears all exercise states back to idle and clears all fields
- Same behavior as current reset but respecting the new state model

## Implementation Notes

### `_ExerciseEntry` Changes

The current `_ExerciseEntry` helper class needs to track state. Add:

```dart
enum ExerciseEntryState { idle, expanded, completed }
```

Add to `_ExerciseEntry`:
- `ExerciseEntryState state` (default: `idle`)
- A method or flag to track whether the exercise should be saved

### `_onBodyPartsChanged()` Changes

Currently this method:
1. Fetches exercises for selected body parts
2. Loads latest weights
3. Creates `_ExerciseEntry` objects with pre-filled controllers
4. Replaces the entire `_exerciseEntries` list

**Change:** When rebuilding the list, preserve completed entries for body parts that are still selected. Only create new entries for exercises that weren't in the previous list.

### `_saveSession()` Changes

Currently saves ALL exercises with non-empty weight. Change to:

```dart
// Only save exercises that were completed (tapped "Train" + "Complete")
for (final entry in _exerciseEntries) {
  if (entry.state != ExerciseEntryState.completed) continue;
  // ... insert WeightTraining
}
```

Also: if an exercise is currently in "expanded" state when save is pressed, treat it as completed.

### `_buildExerciseCard()` Changes

Replace the current always-expanded card with a state-dependent widget:

**Idle state:**
```
┌─────────────────────────────────────────┐
│  Squat                    Last: 40kg     │
│                              [Train]    │
└─────────────────────────────────────────┘
```

**Expanded state:**
```
┌─────────────────────────────────────────┐
│  Squat                    Last: 40kg     │
│  ┌──────────┬────────┬────────┐         │
│  │ Weight   │ Reps   │ Sets   │         │
│  │ 40    kg │ 12     │ 4      │         │
│  └──────────┴────────┴────────┘         │
│                              [Complete] │
└─────────────────────────────────────────┘
```

**Completed state:**
```
┌─────────────────────────────────────────┐
│  ✓ Squat — 40kg × 12 × 4     [Edit]    │
└─────────────────────────────────────────┘
```

### Single-Expand Enforcement

When "Train" is tapped on an idle exercise:
1. Find any currently expanded exercise
2. If found, auto-complete it (save its current field values, set state to completed)
3. Set the tapped exercise to expanded

When "Complete" is tapped:
1. Set the exercise state to completed
2. No other exercise auto-expands

### Edge Cases

- **Empty weight field on complete:** If the user taps "Complete" with an empty weight field, still mark as completed but with empty values. Don't save to WEIGHT_TRAINING on session save (empty weight = skip, same as current behavior).
- **Switching training style mid-session:** The latest weight lookup uses the current training style. If the user toggles between Hyper/Strength, the pre-filled values should update. For v0.1.1, keep it simple: the pre-fill happens when "Train" is tapped, using the style active at that moment. If the user toggles style after expanding, the pre-filled values don't change (they've already started editing).
- **All exercises completed, then body part deselected:** If the user completes exercises, then deselects a body part, those exercises are removed from the list. They are NOT saved (they were never committed to DB). This is consistent — deselecting a body part means "I'm not training that today."
